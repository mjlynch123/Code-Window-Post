A Quick Guide Of How To Use Position

If you are interested in checking out the GitHub repo, click here.

Using the position property can confuse many new coders, but fear not, cause I am here with this walkthrough of how to use this property effectively.

To begin, we will create a new HTML file. Feel free to use your preferred IDE, in this example I will use VSCode. Once we are in our preferred IDE, go ahead and create a file and name it index.html. When that is done, We will create another file and name it index.css.

We've created our HTML and CSS files, we can now begin coding. Within the body of the page, establish a section and give it a class name of top-right, This will be used later. Within this section, create an h1 element. Inside of the newly created h1 element, put your name.

It should look like this:

                
 <!-- Creating a section with a class name of top-right -- >
 <section class="top-right">
    <h1>Your Name</h1>
 </section>

Static

There 5 main position values that we will focus on. The first one is static. When you put your element on the page, static will be the default value. If you are following along you can go ahead and open up your file in a browser of your choice. You should see your name in the top left hand corner of the page. This is what a position of static looks like on your page. NOTE: You do not have to define an element as position: static; unless you are moving that certain element back to default positioning.

Relative

The second value of the five will be relative. This means that the element is relative to it normal position, while also keeping it normal flow within the document. This is best for when you are working with nested elements because it will allow you to move an element around within the parent element and not affect the flow of any others.

Using relative will allow you to use the top, bottom, left, and right properties. These will help you align content within the parent element by giving these values. You can use px, %, em, and rem to put these elements where you want them to be. 

                    
 .top-right {
    border 1px solid black;
    width: 350px;
    }

 .h1 {
    position relative;
    background-color: black;
 }

We have now added styling to our page, achieving a basic shape and visual appearance for our window. Feel free to personalize and tailor the design to your liking. Let's now delve further into the specific functions of each styling element: 

                    
 .code-window {
    box-shadow: -3px 5px 15px #a6a6a6;
    background-color: black;
    color: silver;
    border-radius: 5px;
    margin-left: auto;
    margin-right: auto;;
    width: 800px;
 }

The code-window class selector has been utilized to add a subtle three-dimensional effect through the implementation of a light gray box-shadow, as well as a black background-color. Additionally, a default color of silver has been set for elements without a specified class. To achieve a polished look, a border-radius of 5px has been applied, creating rounded corners for the window. The margin-right and margin-left attributes have been set to auto" to ensure that the window remains centered within the browser window, a best practice for maintaining visual balance. Additionally, a width of 800px has been set for the window. 

                    
 .top-bar {
    height: 20px;
    display: flex;
    align-items: center;
 }

This section may be challenging to understand if you are not familiar with the flex layout. We are establishing a height of 20px for the top-bar div and utilizing flex layout to ensure that the window buttons are aligned horizontally within the div class. The buttons are also centered within the top-bar div. 

                    
 .close, .minimize, .maximize {
    border-radius: 15px;
    margin-inline-start: 7px;
    height: 11px;
    width: 11px;
 }

Here we are grabbing the selectors close, minimize, and maximize and giving them a border-radius of 15px so that they appear circular and then adding a height and width of 11px so that we have a perfect circle. The one thing some may not understand is margin-inline-start, this adds margin to the start of the element, so here we are giving it a value of 7px meaning that at the beginging of the div, there will be 7px from the element in front of it.

Now we get to move onto the fun part where we can actually display our code to the page. Follow along down below to find out more about this: 

                    
 <pre>
    <code>
    </code>
 </pre>

This is the basic setup that you want to start with if you are trying to display code in your window. The first tag that we see is pre. This stands for preformatted text and then we have code. This is where all the code that you want to display goes. Don't forget to add this to the code-window div. Let's move onto the next part. 

                    
 <pre>
    <code>
        &#60;p&#62;display a paragraph element to code&#60;/p&#62;
    </code>
 </pre>

In this article, we learned how to do many things. Firstly, we learned how to use pre and code. Secondly, we learned how to use HTML entities which can display symbols that would otherwise be read as code in the IDE.
